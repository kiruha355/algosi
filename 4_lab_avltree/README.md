## Теория
АВЛ-дерево (AVL Tree)
АВЛ-дерево — самобалансирующееся двоичное дерево поиска, в котором для каждого узла разница высот левого и правого поддеревьев не превышает 1.
balance = height(left) - height(right) in {-1, 0, 1}

### Сложность операций:
```
Операция    Сложность
Вставка     O(log n)
Удаление    O(log n)
Поиск       O(log n)
Память      O(n)
```

Ключи — непустые последовательности латинских букв длиной от 1 до 6 символов. Упорядочиваются лексикографически.

## Решение
 
### Структуры данных
 
Каждый узел дерева хранит ключ, значение, указатели на левое и правое поддеревья, и высоту поддерева с корнем в этом узле:
 
```c
typedef struct AVLNode {
    char key[7];       
    double value;      
    struct AVLNode *left, *right;
    int height;
} AVLNode;
```
 
### Вставка
 
Вставка выполняется рекурсивно как в обычном BST, после чего на обратном ходу рекурсии вызывается `rebalance()`. Если ключ уже существует — значение обновляется.
 
### Удаление
 
При удалении узла с двумя детьми он заменяется **in-order successor** (минимальный элемент правого поддерева). После замены балансировка выполняется снизу вверх.
 
### Балансировка
 
После каждой вставки и удаления вызывается `rebalance()`, которая вычисляет фактор балансировки и выполняет нужный поворот:
 
```
bf = height(left) - height(right)
 
bf > 1  → правый поворот (или левый+правый при bf левого < 0)
bf < -1 → левый поворот  (или правый+левый при bf правого > 0)
```
 
### Повороты
 
**Малый правый поворот** (Right Rotation):
```
    C          B
   /          / \
  B    →     A   C
 /
A
```
 
**Малый левый поворот** (Left Rotation) — симметричен.
 
**Большие повороты** — комбинация двух малых (сначала поворачивается дочерний узел, потом корень).
```
    C          B
   /          / \
  A    →     A   C
   \
    B
 ```
### Split
 
Обходит исходное дерево и вставляет каждый узел в левое или правое результирующее дерево в зависимости от сравнения с pivot:
 
```
key <= pivot → left
key >  pivot → right
```
 
### Merge
 
Обходит оба дерева и вставляет все узлы в результирующее дерево. Балансировка происходит автоматически при каждой вставке.
